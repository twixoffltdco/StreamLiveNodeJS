# StreamLive

Платформа для создания онлайн-телеканалов и радио с модерацией, соц. авторизацией, полноценной админкой, чатом в реальном времени и стикерами.

> Node.js версия. PHP-версия — отдельный репозиторий/поставка (см. раздел ниже).

## Возможности

- **Авторизация и регистрация**: email+пароль, а также любые OAuth2-провайдеры (VK, Google, Яндекс — пресеты из коробки, плюс можно добавить любой другой через `/admin/oauth` без деплоя нового кода).
- **Создание каналов/радио**: владелец указывает название, описание, логотип, источник вещания. Канал уходит на модерацию.
- **Модерация**: админ одобряет или отклоняет канал. При отклонении автору приходит уведомление с причиной, а при попытке зайти на канал зритель видит понятную ошибку "канал не допущен в каталог". Админ может в любой момент снять с публикации уже одобренный канал.
- **SEO**: для каждого канала настраиваются свои `<title>`, meta description и keywords — заточено под индексацию в Яндексе и Google.
- **Плеер**: поддержка MP4, M3U8 (HLS.js), embed YouTube/VK Видео/Rutube и произвольных iframe-источников. Отдельный `/channels/:slug/embed` роут для встраивания на сторонние сайты.
- **Расписание эфира**: понедельно (Пн–Вс), с автоматическим переключением активного источника в реальном времени по текущему времени.
- **Свои источники**: если владелец не хочет использовать стандартные источники платформы — может добавить свои (прямые ссылки, m3u8, встраиваемые платформы).
- **Чат в реальном времени** (Socket.io): назначение модераторов, бан/удаление сообщений, кастомные стикеры канала (как на Twitch).
- **Просмотры**: засчитываются при каждом открытии страницы канала.
- **Дизайн**: тёмная тема в духе TikTok — вертикальные карточки каталога, неоновый акцент.

## Стек

Node.js, Express, EJS, MySQL (mysql2), Passport (local + динамический OAuth2), Socket.io.

## Установка

Есть два способа — рекомендуем веб-мастер, он проще и надёжнее для не-технических админов.

### Вариант A — веб-мастер установки (рекомендуется)

```bash
git clone https://github.com/OinkTechLLC/streamlive.git
cd streamlive
npm install
npm start
```

Откройте `http://localhost:3000` — приложение само определит, что не установлено, и перекинет на `/install`. Мастер из 3 шагов:

1. **Проверка окружения** — Node.js и зависимости.
2. **Проект и база данных** — название проекта, URL, данные MySQL. База данных создаётся автоматически, если её ещё нет.
3. **Аккаунт администратора** — email/логин/пароль. По завершении автоматически заливается схема БД, создаётся администратор, добавляются пресеты соц. провайдеров, записывается `.env`.

После шага 3 перезапустите сервер (`npm start`) — далее приложение работает в обычном режиме.

Если нужно переустановить — удалите `config/installed.json` и `.env`, перезапустите сервер.

### Вариант B — установка через CLI

```bash
cp .env.example .env   # заполните данные MySQL и админа вручную
npm run install-db     # создаёт таблицы и первого администратора без веб-мастера
npm start
```

Затем зайдите в `/admin/oauth`, чтобы подключить VK/Google/Яндекс (впишите свои client_id/secret из кабинетов разработчика).

## Структура проекта

```
streamlive/
├── server.js              # точка входа
├── install.js             # установщик БД + первый админ
├── config/
│   ├── db.js               # пул MySQL
│   └── passport.js         # локальная стратегия + динамические OAuth2
├── middleware/auth.js       # ensureAuth / ensureAdmin / ensureNotBanned
├── routes/
│   ├── install.js           # веб-мастер установки (bootstrap-режим до создания config/installed.json)
│   ├── auth.js              # логин/регистрация/соцвход
│   ├── channels.js          # каталог, дашборд, управление, embed, публичная страница
│   └── admin.js             # модерация, источники, oauth, пользователи
├── sockets/chat.js          # realtime чат, стикеры, баны, удаление сообщений
├── views/                   # EJS-шаблоны (в т.ч. views/admin/*)
├── public/css/style.css     # тёмная тема в стиле TikTok
└── sql/schema.sql           # схема MySQL
```
